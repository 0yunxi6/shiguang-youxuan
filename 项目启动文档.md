# 拾光 - 项目启动文档

## 项目简介

拾光是一个全栈电商平台，采用前后端分离架构：

- **前端**：Vue 3 + Vite + Element Plus
- **后端**：Spring Boot 3.1.5 + MyBatis Plus
- **数据库**：MySQL 8.0
- **缓存**：Redis 7
- **容器化**：Docker & Docker Compose

## 方式一：Docker Compose 一键启动（推荐）

最简单的方式，自动启动所有服务（MySQL、Redis、后端、前端）。

```bash
cd ecommerce-platform
docker-compose up -d
docker-compose up -d --build frontend
```

启动后访问：
- 前端页面：http://localhost
- 后端接口：http://localhost:8080/api

停止服务：
```bash
docker-compose down
```

---

## 方式二：本地开发启动

### 1. 数据库准备

创建 MySQL 数据库并导入初始化脚本：

```sql
CREATE DATABASE ecommerce DEFAULT CHARACTER SET utf8mb4;
```

执行初始化 SQL：
```bash
mysql -u root -p ecommerce < backend/src/main/resources/db/init.sql
```

默认数据库配置（见 `backend/src/main/resources/application.yml`）：
- 地址：localhost:3306
- 用户名：root
- 密码：123456

### 2. 启动后端

```bash
cd ecommerce-platform/backend

# 安装依赖（使用阿里云镜像，已配置 settings.xml）
mvn clean install -s settings.xml

# 启动
mvn spring-boot:run
```

后端启动在 http://localhost:8080

### 3. 启动前端

```bash
cd ecommerce-platform/frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端启动在 http://localhost:5173

## 环境变量配置

### 前端

| 文件 | 变量 | 说明 |
|------|------|------|
| `.env.development` | `VITE_API_BASE_URL` | 开发环境 API 地址，默认 http://localhost:8080 |
| `.env.production` | `VITE_API_BASE_URL` | 生产环境 API 地址，默认 /api |

### 后端（Docker 环境）

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `SPRING_DATASOURCE_URL` | 数据库连接 | jdbc:mysql://mysql:3306/ecommerce |
| `SPRING_DATASOURCE_USERNAME` | 数据库用户 | root |
| `SPRING_DATASOURCE_PASSWORD` | 数据库密码 | root123456 |
| `SPRING_REDIS_HOST` | Redis 地址 | redis |
| `JWT_SECRET` | JWT 签名密钥 | 自定义（512位以上） |

## 构建部署

### 前端构建

```bash
cd frontend
npm run build
# 产物输出到 dist/ 目录
```

### 后端构建

```bash
cd backend
mvn clean package -s settings.xml
# 产物：target/ecommerce-backend-1.0.0.jar
java -jar target/ecommerce-backend-1.0.0.jar
```

## 常见问题

**Q: Maven 下载依赖慢？**
项目已配置阿里云镜像（`backend/settings.xml`），使用 `-s settings.xml` 参数即可。

**Q: 前端接口请求 404？**
确认后端已启动，检查 `.env.development` 中的 API 地址配置。

**Q: 数据库连接失败？**
检查 MySQL 是否启动，确认 `application.yml` 中的数据库地址、用户名、密码是否正确。
