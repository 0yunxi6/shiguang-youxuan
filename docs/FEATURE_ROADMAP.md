# 拾光功能规划文档

本文档用于记录“拾光”后续可增加的功能、优化方向和实施优先级。功能按 **近期可落地**、**中期增强**、**后期高级能力** 三个阶段规划，方便后续持续迭代。

## 优先级说明

| 优先级 | 含义 |
| --- | --- |
| P0 | 影响核心流程或安全，建议优先完成 |
| P1 | 能明显提升用户体验或管理效率，适合近期迭代 |
| P2 | 功能增强型需求，需要一定开发量 |
| P3 | 长期规划，通常依赖第三方服务、AI、支付或复杂业务 |

---

## 二点五、本轮中期基础版落地清单

以下中期功能已在当前代码中完成“可运行基础版”，复杂的第三方或 AI 能力仍保留在后期阶段：

- **会员账号**：会员等级、积分到账、积分兑换优惠券、消息中心、登录通知、邮箱验证模拟流程、两步验证演示流程。
- **商品与搜索**：品牌筛选、规格/SKU 文本配置与下单规格记录、评分筛选、浏览历史、猜你喜欢、商品视频、限时活动价。
- **交易下单**：模拟支付、最优优惠券、订单超时自动取消、发票抬头/税号管理、满 99 包邮否则 8 元运费模板。
- **售后评价**：退款/退货/换货申请、售后进度、评价多图、追评、管理员回复评价。
- **营销运营**：优惠券模板、批量发券、新人礼包、商品分享链接、后台销售/热销/库存数据看板。
- **安全管理**：JWT 退出黑名单、密码修改失效当前 token、后台角色分级、用户禁用原因、文件头校验、接口限流。
- **品牌与体验**：全站品牌统一为“拾光”，接入 Logo/默认头像，新增参考图风格右侧悬浮工具栏与 AI 客服工作台。
- **AI 客服基础版**：新增 GLM-5.1 对话接口；用户端 `/service` 用于用户咨询与自助查询，后台 `/admin/service` 用于客服坐席工作台；可读取当前用户订单/优惠券/售后上下文，本地兜底保证无 Key 时仍可使用。

## 五、当前优先推荐开发清单

结合当前项目状态，建议下一批优先做：

1. **AI 客服知识库**：把平台规则、售后政策、常见问题沉淀为可维护知识库。
2. **人工客服坐席/工单**：AI 无法解决时转人工，记录处理进度和满意度。
3. **真实物流查询**：接入快递鸟、快递100等平台，补齐订单履约体验。
4. **对象存储**：将商品图、评价图、视频迁移到 R2/OSS/COS/S3。
5. **真实支付回调**：接入微信/支付宝，完成支付验签、退款和对账。
6. **活动数据统计**：补充优惠券、满减、限时活动的领取/使用/转化分析。
7. **风控检测**：识别刷单、恶意注册、异常登录和高频爬虫。
