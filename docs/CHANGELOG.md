# 变更记录

## 2026-04-26

本轮按“先做当前能落地的，复杂功能放后期”的原则，完成一批低风险优化。

### 用户端

- 订单增加状态筛选。
- 订单支持模拟支付、取消订单、确认收货。
- 订单卡片展示商品金额、优惠、运费、实付等价格明细。
- 结算页接入后端收货地址接口，并保留本地地址作为兜底。
- 个人中心新增收货地址管理：新增、编辑、删除、设为默认。
- 商品详情新增后端相关推荐接口调用，并保留同分类商品兜底推荐。
- 首页新增“最近浏览”模块。
- 搜索页支持把关键词、排序、价格区间、仅看有货、页码同步到 URL，刷新和分享链接时条件不会丢失。

### 管理端

- 商品管理新增批量上架、批量下架。
- 商品管理新增库存预警筛选。
- 商品管理新增当前页 CSV 导出。
- 商品创建/编辑增加后端基础校验：名称、分类、价格、库存、状态。
- 订单管理新增批量发货、批量完成、批量取消。
- 订单管理支持按订单号、收货人、手机号搜索。
- 订单详情弹窗补充商品明细和费用明细。
- 订单管理新增当前页 CSV 导出。
- 后台写操作新增审计日志，记录管理员、请求方法、路径和 IP。

### 后端与安全

- 新增收货地址表和 `/addresses` 接口。
- 新增商品相关推荐接口 `/products/{id}/recommendations`。
- 登录失败 5 次后锁定 10 分钟，降低暴力破解风险。
- 业务异常现在会尽量返回匹配的 HTTP 状态码，例如 429、403、404。
- 管理端取消订单时会归还库存，并恢复已使用优惠券。
- 后端新增收货地址单元测试。
